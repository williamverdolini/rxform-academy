How to test a component with reactive forms

Rendering the Component: The render function from @testing-library/angular is used to render the FormValidationsComponent in each test. This allows for interaction with the component as if it were in a real application.

Initial State Verification: Tests verify the initial state of the form, ensuring that only specific fields are present and others are not.

Form Interaction: User interactions are simulated using fireEvent and userEvent to test how the form responds.

Form Value and Validity Checks: The form's values and validity are checked after interactions. For instance, ensuring that the form's value updates correctly when fields are filled and that the form's validity status changes appropriately.

Custom Validation: Custom validation logic is tested, such as checking that the "To Date" is not earlier than the "From Date" and that the "Nickname" is unique. Errors are verified using form.hasError and form.getError.

Error Messages and Alerts: Tests check for the presence of error messages and alerts when the form is invalid.
